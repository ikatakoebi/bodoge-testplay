# Bodoge TestPlay 仕様書

## 1. サービス概要

ボードゲームデザイナー向けのオンラインテストプレイプラットフォーム。
CSV/スプレッドシートからゲームのコンポーネント（カード等）を自動生成し、
リアルタイムで複数人がオンラインプレイできる環境を提供する。

### コンセプト
- **画像不要**: テキストデータ（CSV）だけでカードやコンポーネントが自動生成される
- **即時反映**: データを書き換えたら即座にプレイ画面に反映される
- **汎用性**: ルール処理は人間が行い、コンポーネントの操作に特化する
- **テストプレイ特化**: 初期配置のワンクリックリセット、メモ機能など

### 対象ユーザー
- ボードゲームデザイナー（テストプレイ用途）

### MVPスコープ
- カードゲームに特化（トークン・ボード・タイル・ダイスは後続で対応）

## 2. コンポーネント

### 2-1. カード

CSVからカードを自動生成する。サービスの中核機能。

#### CSV形式

```csv
id, name, type, cost, text, count, color, image_url, min_players, tag
001, 兵士, ユニット, 3, "攻撃力+2", 4, red, , ,
002, 城壁, 建物, 5, "防御力+3", 2, blue, https://..., ,
003, 大将軍, ユニット, 8, "全体+3", 1, red, , 4, 5p_only
```

| カラム | 説明 | 必須 |
|--------|------|------|
| id | カードの一意識別子 | Yes |
| name | カード名 | Yes |
| type | カードの分類（自由定義） | No |
| cost | コスト値 | No |
| text | カードのテキスト/効果説明 | No |
| count | そのカードの枚数（デフォルト1） | No |
| color | カードの色/ボーダー色 | No |
| image_url | カード内に表示する画像URL | No |
| min_players | このカードを使う最小プレイ人数 | No |
| tag | セットアップ時のフィルタ用タグ | No |

※ カラムはゲームに応じて自由に追加可能。テンプレートで表示項目を指定する。

#### カードテンプレート

カードのサイズ・レイアウトを定義する。デフォルトテンプレートを用意しつつカスタマイズ可能。

```yaml
card_template:
  size:
    preset: standard       # standard(63x88mm) / mini / tarot / custom
    # custom の場合:
    # width: 50mm
    # height: 70mm

  layout:
    - field: "name"
      position: top
      font_size: 14
      bold: true
    - field: "cost"
      position: top-right
      shape: circle
      bg_color: gold
    - field: "image"
      position: center
      height: 40%
    - field: "type"
      position: center-left
      font_size: 10
      italic: true
    - field: "text"
      position: bottom
      font_size: 10

  back:
    bg_color: "#2a2a5a"
    text: "★"
    # または画像:
    # image_url: "back.png"

  border:
    color: "{{color}}"      # CSVのcolorカラムから動的適用
    radius: 8px
```

#### カードの表/裏と所有権

カードには表面と裏面がある。

- **裏面**: 全プレイヤーに共通のデザインが見える
- **表面**: 所有者（めくった人）だけが見える
- **オープン状態**: 全員に表面が見える

**所有権ルール:**
- カードをめくった人が所有者になる
- 所有者だけがそのカードの表面を閲覧できる
- 右クリックメニューで状態を変更する

#### カードの右クリックメニュー

```
┌────────────────────┐
│ めくる（自分だけ）   │  → 所有者になり表面を確認
│ オープンにする       │  → 全員に表面を公開
│ 裏に戻す            │  → 裏面にして所有権リセット
│ 所有権を渡す    ▶   │  → 他プレイヤーを選択
│ ──────────────────  │
│ 一番上に移動        │
│ ロック/解除         │
└────────────────────┘
```

#### 自動スタック（山札化）

同じサイズのカードを重ねると自動的に山札になる。

**スタック条件:**
- 同サイズのカード同士
- カードの中心同士が一定距離以内に配置された場合

**山札の表示:**
- 枚数を表示
- 一番上のカードの表裏状態を反映

**山札の右クリックメニュー:**
```
┌────────────────────────┐
│ 1枚引く                 │
│ N枚引く...              │
│ シャッフル              │
│ 一番上を見る（自分だけ）  │
│ 全部広げる              │
│ オープンにして広げる      │
└────────────────────────┘
```

### 2-2. カウンター

プレイ中に自由に生成・配置できる数値カウンター。

**特徴:**
- フィールド上に自由に配置・ドラッグ移動可能
- プレイ中にいくつでも生成できる
- 名前・数値をダブルクリックで編集
- [−] / [＋] ボタンで増減
- 右クリックで複製・削除
- 誰でも操作可能

**表示:**
```
┌─────────────┐
│  HP          │  ← ダブルクリックで名前編集
│ [−]  3  [＋] │
└─────────────┘
```

**CSV事前定義（任意）:**
```csv
id, name, min, max, default, step, per_player
hp, 体力, 0, 20, 10, 1, true
round, ラウンド, 1, 10, 1, 1, false
gold, 所持金, 0, 999, 0, 1, true
```

- `per_player: true` の場合、セットアップ時に各プレイヤーに自動生成
- CSVで定義しなくても、プレイ中にフィールド右クリックから自由に生成可能

### 2-3. 画像オブジェクト

フィールド上に画像を自由に配置できる。

**配置方法:**
- ファイルをフィールドにドラッグ&ドロップ
- 右クリックメニュー → 「画像を置く」→ ファイル選択

**操作:**
- ドラッグで移動
- リサイズハンドルで拡大縮小
- 右クリックで削除
- 誰でも操作可能

## 3. フィールド

### 3-1. グリッド / フリー切り替え

フィールドの配置モードをトグルで切り替える。

**グリッドモード:**
- マス目が表示される
- カードやコンポーネントがマスにスナップする
- グリッドサイズはカードサイズに合わせて自動調整 or 手動設定

**フリーモード:**
- 完全自由配置
- スナップなし

**切り替え:**
- 画面上部のトグルボタンで即時切り替え
- グリッドモード中もスナップを無視した配置は可能（Shiftキー等）

### 3-2. エリアシステム

フィールド上に名前付きの矩形エリアを定義できる。
初期配置やゾーン管理に利用する。

**双方向設計:**
- **サービスUI → CSV**: フィールド上で矩形をドラッグしてエリア作成 → 「コピー」ボタンでCSV形式テキストを取得
- **CSV → サービスUI**: CSVをアップロードするとフィールド上にエリアが再現される

**エリアCSV形式:**
```csv
area_id, name, x, y, width, height, visibility, per_player
deck, 山札, 0, 0, 2, 3, hidden, false
market, 市場, 3, 0, 5, 2, public, false
p_field, プレイヤーの場, 0, 4, 8, 2, public, true
p_hand, 手札, 0, 6, 6, 1, owner, true
```

| カラム | 説明 |
|--------|------|
| area_id | エリアの一意識別子（セットアップから参照用） |
| name | 表示名 |
| x, y | グリッド上の座標（左上基準） |
| width, height | エリアのサイズ（グリッド単位） |
| visibility | public / owner / hidden |
| per_player | true の場合プレイヤーごとに生成 |

**エリアのプロパティ（UI上で編集）:**
- 名前
- ID
- 可視性
- per_player フラグ
- 背景色（任意）

**フィールド上の表示:**
```
┌──────────────────────────────────────┐
│                                      │
│   ┌─ 山札 ────────┐  ┌─ 市場 ──────┐ │
│   │ (hidden)      │  │ (public)    │ │
│   │               │  │              │ │
│   └───────────────┘  └──────────────┘ │
│                                      │
│   ┌─ P1の場 (public) ────────────┐   │
│   │                               │   │
│   └───────────────────────────────┘   │
│   ┌─ P2の場 (public) ────────────┐   │
│   │                               │   │
│   └───────────────────────────────┘   │
└──────────────────────────────────────┘
```

## 4. プレイヤーアクション

### 4-1. 操作権限

**基本方針: 全員が全部触れる**

- 操作権限の制限は基本なし
- 誰でも何でも動かせる（ルール管理は人間が行う）
- 唯一の制約はカード表面の可視性（所有者のみ閲覧可）

### 4-2. 操作一覧

| 操作 | 対象 | 方法 |
|------|------|------|
| 移動 | カード/カウンター/画像 | ドラッグ&ドロップ |
| めくる | カード | 右クリックメニュー |
| オープン | カード | 右クリックメニュー |
| シャッフル | 山札 | 右クリックメニュー |
| ドロー | 山札 | 右クリックメニュー → N枚引く |
| 増減 | カウンター | [−] / [＋] ボタン |
| リサイズ | 画像 | リサイズハンドル |
| 回転 | カード/画像 | 右クリックメニュー（将来拡張） |
| ロック | 任意 | 右クリックメニュー → 位置固定 |
| グループ選択 | 複数コンポーネント | 範囲ドラッグで選択 → まとめて移動 |

### 4-3. Undo / Redo

- 操作ログベースで直近の操作を巻き戻し/やり直し
- Ctrl+Z / Ctrl+Y（またはCmd+Z / Cmd+Shift+Z）
- 全プレイヤーの操作が対象（誰の操作でも戻せる）
- 操作ログとして記録される項目:
  - コンポーネントの移動
  - カードのめくり/オープン
  - カウンターの増減
  - コンポーネントの生成/削除

## 5. 初期配置（セットアップ）システム

### 5-1. プレイ人数設定

```yaml
game:
  players:
    min: 2
    max: 5
```

- ルーム作成時にホストがプレイ人数を設定
- 参加者が揃った時点で確定も可

### 5-2. セットアップ定義

人数に応じた初期配置をYAMLで定義する。

```yaml
setup:
  # 人数によってカードを除外
  - action: remove
    from: deck
    filter: { tag: "5p_only" }
    when: "players < 5"

  # 山札シャッフル
  - action: shuffle
    target: deck

  # 各プレイヤーに手札を配る（人数で枚数変動）
  - action: deal
    from: deck
    to: p_hand            # エリアIDで指定
    per_player: true
    count:
      2p: 7
      3p: 6
      4p: 5
      5p: 4

  # 場にカードをオープン
  - action: deal
    from: deck
    to: market
    face_up: true
    count:
      2p: 4
      3p: 5
      4p: 6
      5p: 6

  # 各プレイヤーに初期リソース（カウンター自動配置）
  - action: give
    component: gold_token
    to: p_field
    per_player: true
    count:
      default: 3
      2p: 5
```

### 5-3. ワンクリックリセット

- 「リセット」ボタンで初期配置に即座に戻る
- 同じ人数設定でセットアップを再実行
- シャッフルは毎回ランダム

## 6. CSVホットリロード

### 仕様

- CSVを再アップロードするとプレイ中でも即座に反映
- **配置やゾーン状態は保持したまま**、データ（テキスト、コスト等）だけ差し替え
- 例: 「カード003のコスト3→4に変更」→ 手札にあるそのカードも即座にコスト4に更新

### 差分検出

- 既存カードの変更 → 即時反映（配置はそのまま）
- カードの追加 → 追加分は山札等の指定エリアに配置（または未配置エリアに）
- カードの削除 → 警告を表示（盤面上にある場合は確認ダイアログ）
- カードの枚数変更 → 増減を反映、減った分は盤面上のものから除去（確認あり）

### 操作フロー

```
1. スプレッドシートでデータ編集
2. CSVとしてダウンロード
3. サービス画面にドラッグ&ドロップ（または再アップロードボタン）
4. 差分を検出して即座に反映
5. 変更サマリーをトースト通知で表示
```

## 7. リアルタイム通信

### アーキテクチャ

```
┌──────────┐    WebSocket     ┌──────────┐
│ Player A │◄───────────────►│  Server  │
└──────────┘                  │          │
┌──────────┐    WebSocket     │  状態管理 │
│ Player B │◄───────────────►│          │
└──────────┘                  │          │
┌──────────┐    WebSocket     │          │
│   ...    │◄───────────────►│          │
└──────────┘                  └──────────┘
```

### ルーム

- ルーム作成 → 招待URL発行
- URLを知っていれば誰でも参加可能（人数制限なし）
- 参加者は名前と色を選択
- ホスト権限: ゲーム設定変更、リセット実行

### 同期対象

- コンポーネントの位置・状態変更
- カードのめくり/オープン状態
- カウンターの値変更
- コンポーネントの生成/削除
- Undo/Redo操作
- 操作ログ

### アクセス制限

- なし。URL共有ベース。
- 全員が同じ操作権限を持つ。

## 8. 操作ログ

プレイ中の全操作をログとして記録・表示する。

**表示場所:** サイドパネル（常時表示 or トグル）

**ログ例:**
```
[14:32:05] Aが山札から1枚引いた
[14:32:12] Bがカード「兵士」をオープンにした
[14:32:18] Aが所持金を 3 → 0 に変更
[14:32:25] Cがダイスを振った: 5, 3（将来機能）
```

## 9. テストプレイメモ

デザイナー向け機能。プレイ中に気づいたことをメモとして記録する。

**機能:**
- フィールド上に付箋（メモ）を配置できる
- テキスト入力
- ドラッグで移動可能
- 色分け（重要度やカテゴリ）
- メモ一覧をエクスポート可能（テストプレイ後の振り返り用）
- 誰が書いたかを記録

## 12. ゲーム定義ファイル構成

```
my_game/
├── game.yaml              # メタ情報（ゲーム名、人数設定、グリッド設定）
├── cards.csv              # カードデータ
├── areas.csv              # エリア定義
├── counters.csv           # カウンター事前定義（任意）
├── setup.yaml             # 初期配置＋人数別設定

├── templates/
│   ├── card_default.yaml  # デフォルトカードテンプレート
│   └── card_spell.yaml    # タイプ別テンプレート（任意）
└── assets/                # 画像ファイル（任意）
    ├── card_back.png
    └── card_art/
```

### game.yaml 例

```yaml
game:
  name: "カードバトルα"
  version: "0.3"
  players:
    min: 2
    max: 5
  field:
    grid:
      enabled: true
      cols: 12
      rows: 8
      cell_size: 80px
    background_color: "#2d5a3d"
```

## 13. 技術スタック（案）

| レイヤー | 技術 | 理由 |
|----------|------|------|
| フロントエンド | React + TypeScript | コンポーネント指向、エコシステム |
| 描画 | HTML/CSS + Canvas (Konva.js等) | カード描画はDOM、自由配置はCanvas |
| リアルタイム通信 | WebSocket (Socket.io) | 双方向リアルタイム同期 |
| バックエンド | Node.js (Express or Fastify) | フロントとJS統一 |
| 状態管理 | Zustand | 軽量でゲーム状態管理に適する |
| DB | PostgreSQL + Redis | 永続化 + リアルタイムセッション管理 |
| CSV解析 | Papa Parse | CSV/TSVの読み込み |
| ホスティング | Vercel (Front) + Railway/Fly.io (Back) | 候補 |
